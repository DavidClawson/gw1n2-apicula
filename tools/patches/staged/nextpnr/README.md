# Staged nextpnr GW1N-2 patch (YosysHQ/nextpnr)

Separate repo from apicula. Reviewer: `gatecat` (himbaechel/IO side).
Base commit: `32324500` (the commit the oss-cad-suite binary was built from).

## What's here
- `01-gw1n2-arch-gen.patch` — the **entire** nextpnr-side change, 2 files:
  - `himbaechel/uarch/gowin/CMakeLists.txt`: add `GW1N-2` to
    `ALL_HIMBAECHEL_GOWIN_DEVICES` so the build generates `chipdb-GW1N-2.bin`.
  - `himbaechel/uarch/gowin/gowin_arch_gen.py`:
    - add `GW1N-2` to the `simple_io` device set (GW1N-1P5C die uses the same
      small-part IO layout as GW1NZ-1/GW1N-1);
    - in `create_pll_tiletype`, emit the PLL tile **without** a PLL bel when the
      chipdb has no `RPLLA` (GW1N-2's PLL is not yet decoded — see apicula
      `docs/06`/`07`). Logic/routing/IO still build; a design that uses the PLL
      would fail at pack time, which is the intended WIP behaviour.

## What is deliberately NOT here
- **No `gowin.cc` change.** Earlier notes assumed a device→family regex fix was
  needed because the GW1N-1P5C partnumber (`GW1N-UV1P5…`) parses to family
  `GW1N-1`. It isn't: the examples Makefile passes `--vopt family=` explicitly as
  the house idiom (e.g. `tangnano9k` → `--vopt family=GW1N-9C`,
  `tangnano20k` → `--vopt family=GW2A-18C`). The GW1N-2 example does the same
  (`--vopt family=GW1N-2`), so no parser hack is required.

## Build / test (mirrors `apicula/.github/workflows/toolchain.yml`)
The CI builds nextpnr from source with `-DHIMBAECHEL_GOWIN_DEVICES="all"`, which
runs `gowin_arch_gen.py -d <dev>` against the installed apycula wheel to generate
each chipdb at build time. To validate GW1N-2 alone (it builds independently in
the CMake `foreach`):

```bash
# apycula with GW1N-2 support must be importable; point cmake at it:
export APYCULA_INSTALL_PREFIX=/path/to/apicula/.venv
cd nextpnr
git apply /path/to/01-gw1n2-arch-gen.patch
cmake -B build -DBUILD_PYTHON=OFF -DARCH=himbaechel \
      -DHIMBAECHEL_UARCH=gowin -DHIMBAECHEL_GOWIN_DEVICES="GW1N-2"
make -C build -j$(nproc)          # generates chipdb-GW1N-2.bin + builds the binary
```

Then the example (in the apicula repo):
```bash
cd apicula/examples/gw1n2
make NEXTPNR=/path/to/nextpnr/build/nextpnr-himbaechel gw1n2
```

## Dependency / ordering note for the PR
The nextpnr GW1N-2 chipdb is generated by `gowin_arch_gen.py` reading the apycula
GW1N-2 chipdb, so the apicula PRs (PR1 partType-1 `.dat`, PR2 recognize+build,
PR3 pinout/IO) must land (or be installed) first. Coordinate the two repos with
`yrabbit` (apicula) and `gatecat` (nextpnr).
